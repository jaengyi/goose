자율 에이전트로서 goose는 지정된 지시사항에 따라 작업을 수행하도록 설계되었습니다. 이는 종종 로컬 파일 작업을 포함합니다 - 작업할 올바른 파일을 찾고 안전하게 수정하는 것 모두를 말합니다.

이 가이드는 goose에서 파일에 효율적으로 액세스하고 참조하는 방법을 다룹니다. 또한 변경 사항을 모니터링하고 필요할 때 되돌리는 것과 같은 안전한 파일 작업을 위한 필수 모범 사례를 포함하여 코드베이스의 무결성을 유지합니다.

## 파일 액세스

### goose 데스크톱에서 빠른 파일 검색

goose 데스크톱에는 파일 시스템 대화 상자를 수동으로 탐색하지 않고도 채팅 인터페이스 내에서 파일을 쉽게 참조할 수 있는 퍼지 파일 검색 기능이 포함되어 있습니다. 이 기능은 goose에게 보내는 메시지에 파일을 빠르게 찾아 포함하는 데 도움이 됩니다.

1. 채팅 입력창에 `@`를 입력하여 파일 검색 상자를 엽니다
2. 계속 입력하여 대소문자를 구분하지 않는 퍼지 매칭으로 파일을 필터링합니다 (예: `@readme`, `@config.js`, `@src/main`)

   결과 탐색:
   - 화살표 키(↑/↓)를 사용하여 검색 결과를 이동합니다
   - 클릭하거나 `Enter`를 눌러 선택한 파일 경로를 메시지에 삽입합니다

3. 끝! 준비되면 goose에게 메시지를 보내세요

:::info
파일을 선택하지 않고 검색 상자를 닫으려면 `Esc`를 누르거나 채팅 입력창을 클릭하세요.
:::

**스마트 기능:**
- **퍼지 매칭**: 부분 텍스트를 지능적으로 매칭하고 단어 경계에서 일치 항목의 우선순위를 지정합니다
- **강조 표시된 결과**: 검색 결과에서 일치하는 문자를 강조 표시합니다
- **성능 최적화**: 지능적인 필터링으로 최대 5개 디렉토리 레벨까지 스캔합니다
- **자동 필터링**: `.git`, `node_modules`, `__pycache__`, `target`, `dist`, `build`와 같은 일반적인 디렉토리를 자동으로 제외합니다
- **숨김 폴더 지원**: `.github`, `.vscode`, `.idea`, `.config` 및 기타 CI/CD 폴더(`.circleci`, `.gitlab`, `.azure`, `.jenkins`)와 같은 중요한 구성 디렉토리를 포함합니다
- **크로스 플랫폼**: 사용자 디렉토리에서 검색합니다(macOS의 `/Users`, Windows의 `C:\Users`, Linux의 `/home`)
- **시각적 표시**: 명확한 아이콘으로 파일과 디렉토리를 구분합니다

## 파일 관리 모범 사례

### 버전 관리

항상 Git과 같은 버전 관리 시스템을 사용하여 코드베이스의 변경 사항을 추적하세요. 이렇게 하면 실수로 덮어쓰는 것을 방지하고 이전 상태로 쉽게 되돌릴 수 있습니다. 코드베이스에서 goose를 실행하기 전에 변경 사항을 커밋하세요. 브랜치를 사용하여 실험적인 변경 사항을 메인 코드베이스와 분리하세요.

### 검증 및 테스트

goose가 파일을 수정하기 전후에 검증 및 테스트 단계를 구현하세요. 단위 테스트를 실행하여 goose가 수행한 변경 사항을 확인하세요. 변경 사항이 전체 시스템과 잘 통합되는지 확인하기 위해 스테이징 환경을 사용하세요.

### 변경 검토

생성된 코드나 변경 사항의 품질을 보장하기 위해 수동 검토 또는 자동화된 코드 리뷰를 사용하세요. goose가 수행한 변경 사항을 시각화하기 위해 diff 도구와 같은 도구를 통합하세요. 팀 멤버 또는 CI/CD 파이프라인과 함께 검토 프로세스를 구현하세요.

### 코드베이스 조직

코드베이스를 잘 정의된 모듈이나 하위 디렉토리로 구조화하여 효율적으로 관리하세요. 모듈식 접근 방식을 사용하여 goose가 액세스해야 하는 코드 부분을 분리하세요. goose가 작업할 특정 디렉토리나 파일 경로를 제공할 수도 있습니다.
