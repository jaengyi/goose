# 용어집

goose 문서에서 사용되는 주요 용어와 개념을 정리한 목록입니다. 알파벳순 및 한글순으로 정렬되어 있습니다.

---

## 영문 용어

### A

#### Activities (활동)
레시피에서 정의되는 클릭 가능한 예시 프롬프트 버블입니다. goose 데스크톱에서 사용자가 빠르게 작업을 시작할 수 있도록 도와줍니다.

#### Agent (에이전트)
사용자를 대신하여 자율적으로 작업을 수행하는 AI 시스템입니다. goose는 엔지니어링 작업을 자동화하는 로컬 AI 에이전트입니다.

#### Auto Mode (자동 모드)
goose가 사용자의 승인 없이 대부분의 도구를 자동으로 실행할 수 있는 권한 모드입니다.

### C

#### CLI (Command Line Interface)
명령줄 인터페이스. 터미널에서 텍스트 명령을 통해 goose와 상호 작용하는 방식입니다.

#### Code Mode (코드 모드)
MCP 도구를 직접 호출하는 대신 프로그래밍 방식으로 상호 작용하는 모드입니다. 많은 확장 기능이 활성화된 경우 컨텍스트 윈도우를 효율적으로 관리합니다.

#### Compaction (압축)
컨텍스트 윈도우가 가득 찼을 때 이전 대화를 요약하여 새로운 컨텍스트 공간을 확보하는 프로세스입니다.

#### Context Engineering (컨텍스트 엔지니어링)
goose가 효과적으로 작동하도록 배경 지식, 선호도 및 워크플로우를 구축하는 기술입니다. .goosehints, 스킬, 메모리 등을 활용합니다.

#### Context Window (컨텍스트 윈도우)
LLM이 한 번에 처리할 수 있는 텍스트(토큰)의 최대 양입니다. 대화가 길어지면 이 제한에 도달할 수 있습니다.

### D

#### Deeplink (딥링크)
goose 데스크톱에서 특정 레시피를 직접 열 수 있는 URL 링크입니다.

#### Desktop (데스크톱)
goose의 그래픽 사용자 인터페이스(GUI) 애플리케이션입니다. Electron 기반으로 구축되었습니다.

### E

#### Extension (확장 기능)
goose에 기능을 추가하는 패키지입니다. 예: Developer, Google Drive, GitHub 등. 각 확장 기능은 하나 이상의 도구를 포함합니다.

### F

#### Fallback (폴백)
리드/워커 모드에서 워커 모델이 실패할 때 리드 모델로 자동 전환되는 메커니즘입니다.

### G

#### goose
Block이 개발한 로컬, 확장 가능, 오픈소스 AI 에이전트입니다. 엔지니어링 작업을 자동화합니다. 항상 소문자로 표기합니다.

#### .goosehints
프로젝트에 대한 추가 컨텍스트와 지침을 goose에게 제공하는 텍스트 파일입니다. 전역 또는 프로젝트별로 저장할 수 있습니다.

#### .gooseignore
goose가 무시해야 할 파일이나 디렉토리를 지정하는 파일입니다. .gitignore와 유사한 형식을 따릅니다.

### H

#### Headless Mode (헤드리스 모드)
사용자 상호 작용 없이 자동으로 실행되는 goose의 비대화형 모드입니다. CI/CD 파이프라인, 서버 환경, 배치 처리에 적합합니다.

#### Hints (힌트)
.goosehints 파일에 포함된 지침이나 컨텍스트를 의미합니다.

### I

#### Instructions (지침)
레시피에서 에이전트의 동작과 역할을 정의하는 텍스트입니다.

### L

#### Lead Model (리드 모델)
리드/워커 구성에서 초기 계획과 복잡한 추론을 담당하는 더 강력한 AI 모델입니다.

#### Lead/Worker Mode (리드/워커 모드)
두 개의 다른 AI 모델을 페어링하는 구성입니다. 리드 모델이 계획을 세우고 워커 모델이 실행합니다.

#### LLM (Large Language Model)
대규모 언어 모델. GPT-4, Claude 등 goose를 구동하는 AI 모델입니다.

### M

#### Manual Mode (수동 모드)
모든 도구 실행 전에 사용자 승인을 요구하는 권한 모드입니다.

#### MCP (Model Context Protocol)
AI 에이전트와 도구 간의 통신을 위한 프로토콜입니다. goose는 MCP를 통해 확장 기능과 상호 작용합니다.

#### MCP Apps
goose 데스크톱 내에서 인터랙티브 UI를 렌더링할 수 있는 확장 기능을 위한 공식 MCP 사양입니다.

#### MCP-UI
텍스트 응답을 그래픽적이고 인터랙티브한 사용자 경험으로 변환하는 확장 기능 표준입니다.

#### Memory Extension (메모리 확장 기능)
goose가 세션 간에 정보를 기억할 수 있게 하는 내장 확장 기능입니다.

### P

#### Parameters (매개변수)
레시피에서 사용자가 실행 시 입력하는 동적 값입니다. `{{ variable_name }}` 구문을 사용합니다.

#### Permission Mode (권한 모드)
goose가 도구를 실행하는 방식을 제어하는 설정입니다. Auto, Smart, Manual 모드가 있습니다.

#### Plan Mode (계획 모드)
복잡한 작업을 실행하기 전에 단계별 계획을 생성하는 모드입니다. `/plan` 명령으로 활성화합니다.

#### Prompt (프롬프트)
사용자가 AI에게 보내는 입력 텍스트 또는 레시피에서 자동으로 시작되는 초기 메시지입니다.

#### Prompt Injection (프롬프트 인젝션)
악의적인 사용자가 AI 시스템의 동작을 조작하려는 보안 공격입니다.

#### Provider (프로바이더)
LLM 서비스를 제공하는 회사 또는 플랫폼입니다. 예: OpenAI, Anthropic, Google, Ollama 등.

### R

#### Recipe (레시피)
지침, 프롬프트, 확장 기능 구성 등을 패키징한 재사용 가능한 goose 설정입니다. YAML 또는 JSON 형식으로 저장됩니다.

#### Recipe Library (레시피 라이브러리)
goose 데스크톱에서 저장된 레시피를 관리하는 공간입니다.

#### RPI (Research, Plan, Implement)
복잡한 작업을 처리하기 위한 컨텍스트 엔지니어링 패턴입니다. 연구 → 계획 → 구현 순서로 진행됩니다.

### S

#### Session (세션)
goose와의 하나의 대화 인스턴스입니다. 메시지 기록, 컨텍스트, 설정을 포함합니다.

#### Skills (스킬)
goose에게 특정 작업을 수행하는 방법을 가르치는 재사용 가능한 지침 세트입니다. SKILL.md 파일로 정의됩니다.

#### Slash Command (슬래시 명령)
`/` 로 시작하는 명령입니다. 내장 명령(예: `/help`, `/compact`)과 사용자 정의 명령이 있습니다.

#### Smart Mode (스마트 모드)
goose가 도구의 위험 수준에 따라 자동으로 승인 여부를 결정하는 권한 모드입니다.

#### Subrecipe (서브레시피)
메인 레시피에서 호출되는 하위 레시피입니다. 복잡한 워크플로우를 모듈화하는 데 사용됩니다.

### T

#### Token (토큰)
LLM이 텍스트를 처리하는 기본 단위입니다. 대략 영어 4글자 또는 한글 1-2글자에 해당합니다.

#### Tool (도구)
확장 기능 내에서 goose가 실행할 수 있는 특정 기능입니다. 예: 파일 읽기, 셸 명령 실행, 스크린샷 캡처 등.

#### Tool Permission (도구 권한)
특정 도구의 실행 방식을 제어하는 설정입니다. "항상 허용", "사전 확인", "절대 금지" 수준이 있습니다.

#### Turn (턴)
하나의 완전한 상호 작용 - 사용자의 프롬프트와 모델의 응답을 합친 것입니다.

### W

#### Worker Model (워커 모델)
리드/워커 구성에서 실제 작업 실행을 담당하는 빠르고 효율적인 AI 모델입니다.

## 관련 기술 용어

### API (Application Programming Interface)
애플리케이션 프로그래밍 인터페이스. 소프트웨어 구성 요소 간의 상호 작용을 정의합니다.

### CI/CD (Continuous Integration/Continuous Deployment)
지속적 통합/지속적 배포. 자동화된 빌드, 테스트, 배포 파이프라인입니다.

### YAML (YAML Ain't Markup Language)
사람이 읽기 쉬운 데이터 직렬화 형식입니다. 레시피 및 구성 파일에 사용됩니다.

### JSON (JavaScript Object Notation)
경량 데이터 교환 형식입니다. 레시피 및 API 응답에 사용됩니다.

### Markdown
경량 마크업 언어입니다. 문서, 힌트 파일, 스킬 정의에 사용됩니다.

### OpenAPI
REST API를 설명하기 위한 사양입니다. goose 서버의 API 정의에 사용됩니다.

### Rust
시스템 프로그래밍 언어입니다. goose의 핵심 에이전트 로직이 Rust로 작성되었습니다.

### TypeScript
JavaScript의 타입이 지정된 상위 집합입니다. goose 데스크톱 UI가 TypeScript로 작성되었습니다.

### Electron
웹 기술을 사용하여 데스크톱 애플리케이션을 만드는 프레임워크입니다. goose 데스크톱에 사용됩니다.
